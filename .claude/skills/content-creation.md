# Content Creation

## 概要

技術コンテンツ作成のガイドライン。技術ブログ記事、チュートリアル、ドキュメント、ソーシャルメディア投稿など、開発者向けコンテンツの企画・執筆・配信のベストプラクティスを提供する。

## 適用場面

- 技術ブログ記事の執筆
- チュートリアル・ハンズオンの作成
- プロジェクトの告知・広報
- ソーシャルメディアでの技術発信

## ベストプラクティス

### 1. 技術ブログ記事

- タイトルは具体的で検索しやすいものにする
- 冒頭で「この記事で学べること」を明示する
- 問題 -> 解決策 -> 結果の構成を基本とする
- コードスニペットは実際に動作するものを掲載する
- 記事の長さは1500-3000語を目安とする
- SEOを意識したキーワードを自然に含める

### 2. チュートリアル作成

- 前提知識・環境を冒頭に明記する
- ステップバイステップで進行する構成にする
- 各ステップの目的と期待結果を説明する
- つまずきやすいポイントに注意書きを入れる
- 完成したコードのリポジトリを提供する
- 段階的に複雑度を上げる構成にする

### 3. ドキュメント型コンテンツ

- ユーザーの習熟度に応じてコンテンツを分ける（入門/中級/上級）
- Getting Started は5分以内に完了できる内容にする
- FAQ形式で頻出の質問に答える
- トラブルシューティングガイドを用意する
- バージョンごとの変更点を明記する

### 4. ソーシャルメディア

- Twitter/X: 280文字で核心を伝える。スレッド形式で詳細を展開する
- 技術的な図やスクリーンショットを添付する
- ハッシュタグは関連性の高いものを2-3個に絞る
- 投稿タイミングはターゲットの活動時間帯に合わせる
- コミュニティへの貢献を意識し、宣伝色を抑える

### 5. コンテンツの品質管理

- 技術的正確性を複数人でレビューする
- コードスニペットは実行して動作確認する
- 誤字・脱字チェックを行う（textlint等のツールを活用）
- 古くなったコンテンツを定期的に更新する
- 読者からのフィードバックを収集し改善に活かす
- アクセス解析で人気コンテンツの傾向を把握する

## チェックリスト

- [ ] ターゲット読者が明確に定義されている
- [ ] タイトルが具体的で魅力的である
- [ ] コードスニペットが実際に動作する
- [ ] 前提条件が明記されている
- [ ] 技術的正確性がレビュー済みである
- [ ] 誤字・脱字がチェック済みである
- [ ] OGP画像やメタデータが設定されている
