# Product Planning

## 概要

プロダクト企画のガイドライン。MVP定義、ユーザーストーリー作成、ロードマップ策定、成功指標の設定を通じて、ユーザー価値を最大化するプロダクト開発を推進する。

## 適用場面

- 新規プロダクトの企画段階
- MVP (Minimum Viable Product) の定義
- 機能の優先順位付け
- ロードマップの策定・見直し

## ベストプラクティス

### 1. MVP定義

- MVPは「最小限の機能で最大の学びを得る」ために設計する
- コア仮説を明確にし、MVPで検証する仮説を絞る
- スコープは「なくても成立するもの」を削ることで決める
- MVPの完成期間は2-4週間を目安とする
- MVPのリリース後にユーザーフィードバックを収集する仕組みを用意する

### 2. ユーザーストーリー

- フォーマット: 「[ペルソナ]として、[機能]を使って、[価値]を得たい」
- 受入条件を具体的かつテスト可能な形式で記述する
- ストーリーはINVEST原則に従う（Independent, Negotiable, Valuable, Estimable, Small, Testable）
- ストーリーマッピングで全体像とリリース計画を可視化する
- 技術タスクもユーザー価値に紐付けて記述する

### 3. ロードマップ策定

- 時間軸は Now / Next / Later の3段階で管理する
- Now: 現在取り組んでいる（1-2週間）
- Next: 次に取り組む予定（2-4週間後）
- Later: 将来的に検討する（1ヶ月以上先）
- ロードマップは固定的な計画ではなく、学びに基づいて更新する
- 依存関係とリスクを可視化する

### 4. 成功指標 (KPI)

- North Star Metric: プロダクト全体の健全性を示す1つの指標を定義する
- 各機能にOKR (Objectives and Key Results) を設定する
- 定量指標: DAU, 継続率, コンバージョン率, NPS等
- 定性指標: ユーザーインタビュー、サポート問い合わせ傾向
- データ収集の仕組みを機能実装と同時に構築する

### 5. 優先順位付けフレームワーク

- ICE Score: Impact x Confidence x Ease で優先度を計算する
- RICE Score: Reach x Impact x Confidence / Effort で比較する
- MoSCoW: Must have, Should have, Could have, Won't have で分類する
- 優先順位は定期的に見直す（スプリント計画時）
- 「やらないこと」を明確にする

## チェックリスト

- [ ] コア仮説が明確に定義されている
- [ ] MVPのスコープが絞られている
- [ ] ユーザーストーリーがINVEST原則に従っている
- [ ] 成功指標が定量的に定義されている
- [ ] ロードマップが作成されている
- [ ] 優先順位付けの根拠が明確である
